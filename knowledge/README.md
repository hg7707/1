# 知识库填写说明

你后续主要维护这个文件夹。每个文件都是 JSON 数组，也就是：

```json
[
  {
    "id": "唯一编号",
    "title": "条目标题",
    "tags": ["关键词1", "关键词2"],
    "content": "你的游戏理解"
  }
]
```

## 推荐填写顺序

1. 先写 `mistakes.json`

把玩家最常见的错误写进去，比如技能空放、替身乱交、奥义没确认、压场过深。

2. 再写 `trainings.json`

每一种错误都配一个训练作业。作业越具体越好，例如“接下来 5 局只记录每次替身原因”。

3. 再写 `mechanics.json`

把底层机制解释清楚，比如替身、技能冷却、奥义点、保护、扫地、抓取。

4. 最后写 `ninjas.json` 和 `matchups.json`

这两类资料更新成本高，建议等通用复盘跑顺以后再大量补。

## 写法建议

- `title` 要像玩家会说的问题，比如“技能空放后继续前压”。
- `tags` 要放玩家描述里可能出现的词，比如“技能空放”“前压”“替身”“奥义”。
- `content` 不要写成百科，写成能指导复盘的判断原则。
- 每条只讲一个问题，别把多个知识点塞在一起。
- 不确定版本的信息，不要写成绝对结论。

## 忍者条目建议字段

```json
{
  "id": "ninja_xxx_001",
  "title": "某忍者基础打法",
  "ninja": "某忍者",
  "level": "intermediate",
  "tags": ["起手", "强势距离", "弱点"],
  "summary": "这个忍者强在哪里、怕什么、常见起手是什么、技能空了应该怎么处理。"
}
```

## 对局条目建议字段

```json
{
  "id": "matchup_xxx_vs_yyy_001",
  "title": "某忍者打某忍者",
  "playerNinja": "我方忍者",
  "enemyNinja": "对手忍者",
  "level": "advanced",
  "tags": ["matchup", "距离", "反制"],
  "summary": "这组对局的关键距离、不能乱交的技能、推荐争夺节奏。"
}
```

## 注意

带 `"template": true` 的条目只是模板，不会进入复盘检索。你复制模板后，要删除这一行。
